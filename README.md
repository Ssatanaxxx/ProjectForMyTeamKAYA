# Budgetly — автопланирование бюджета и контроль

Веб-приложение для экономиста компании и руководителей департаментов: экономист
задаёт общий бюджет и период, система автоматически распределяет лимиты по
департаментам, руководители формируют заявки (позиция × количество × цена +
обоснование), а экономист проверяет их через автоконтроль, формулы и ИИ-разбор
и утверждает или возвращает на доработку — с уведомлениями в реальном времени.

## Стек

React 19 + TypeScript + Vite, Tailwind CSS, Motion (motion.dev), Recharts,
lucide-react, Supabase (Postgres + Realtime), опционально OpenAI.

## Запуск

```bash
npm install
npm run dev
```

Откройте `http://localhost:5173`.

### Демо-режим (без бэкенда)

Если `.env.local` пустой, приложение работает на `localStorage` с синхронизацией
между вкладками через `BroadcastChannel` — можно пройти весь флоу (экономист +
руководитель департамента) в двух вкладках одного браузера без какой-либо
настройки.

### Реальный бэкенд (Supabase)

1. Создайте проект в [Supabase](https://supabase.com).
2. В SQL Editor выполните `supabase/migrations/001_init.sql` — создаст таблицы,
   статусную машину заявки, view `request_totals`, RLS и включит Realtime.
3. Скопируйте `Project URL` и `anon public` ключ в `.env.local`:

   ```
   VITE_SUPABASE_URL=...
   VITE_SUPABASE_ANON_KEY=...
   ```

4. Перезапустите `npm run dev`. Данные и уведомления теперь идут через реальную
   БД и работают между разными устройствами.

### ИИ-разбор заявок (опционально)

Добавьте `VITE_OPENAI_API_KEY` в `.env.local` (модель по умолчанию —
`VITE_OPENAI_MODEL=gpt-4o-mini`). Без ключа автоконтроль работает на
детерминированных эвристиках (`src/lib/advisor.ts`) — ИИ добавляет только
текстовый разбор поверх и не принимает решения за экономиста.

## Проверка

```bash
npm run test    # юнит-тесты формул, статусной машины и советника (vitest)
npm run build   # проверка типов + сборка
```

## Структура

```
src/
  lib/
    formulas.ts     — формулы экономиста (variance, utilization, burn rate, автораспределение)
    status.ts        — статусная машина заявки
    advisor.ts        — детерминированный автоконтроль (эвристики)
    aiAdvisor.ts       — ИИ-разбор поверх эвристик (OpenAI)
    repo/              — доступ к данным: local (демо) и supabase (прод)
  pages/               — экраны: вход, lobby экономиста, кабинет департамента, разбор, аналитика
  components/          — UI-кит, графики, AppBar, Toast, тема
supabase/migrations/    — SQL-схема
```

## Важное про демо-режим

RLS-политики в миграции открытые (`using (true)`) — это упрощение для хакатона,
изоляция данных обеспечивается на уровне приложения по коду компании. Для
продакшена политики нужно ужесточить под `auth.uid()`.
